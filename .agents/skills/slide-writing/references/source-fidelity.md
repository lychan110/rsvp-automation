\# Source-of-Truth Grounding Rules



Read this file when the user provides a proposal, spec, or other source document to ground slides in.



\## Rules



1\. When the user provides a source-of-truth document, perform a full sweep and ground all slide claims in that document.

2\. If needed to improve narrative quality, pull additional relevant details from the same source-of-truth document rather than adding unsupported claims.

3\. Prefer exact terminology and boundaries from the source document for capabilities, risks, and ownership split.

4\. Do not add behind-the-scenes framing text to presentation slides (e.g., "proposal grounding", "this deck is grounded in", or authoring-process explanations).



\## Full Reset Rewrite Mode



1\. If the user asks to "scrap" slides or diagrams, rebuild the deck and diagram catalog from scratch.

2\. Do not retain old slide framing, section labels, or prior diagram concepts unless explicitly requested.

3\. Re-anchor every slide to the current source document before writing new copy.
